    基础语句·

    1. select & from
    2. select 字段（展示） from 表（数据来源）
    3. * 所有列
    4. as 别名
    5. distinct 去重，必须置首
    6. 计算（必须包含被引用的字段，除非可以独立计算）

    7. select gdp, distinct population, gdp/population as averGDP from world

    8. where

    9. 筛选行数据
    10. where 字段名 运算符 值
    11. 比较符 =, >, <, >=, <=, <>, !=
    12. 条件范围 in (...), not in, between and（闭区间）, is null, is not null
    13. 逻辑 and, or, not
    14. 模糊查询 like 通配符 字符，通配符 %, _ 匹配单个或多个字符

    15. where population >= 20, name in ('Germany', 'Norway')

    16. order by

    17. 规定结果集显示顺序
    18. asc, desc 默认升序
    19. 主字段

    20. 将某些排最后 order by subject in ('chem','phys'), subject

    21. limit

    22. limit [位置偏移量,] 行数

    23. 返回排名第4-7的数据 order by area desc limit 3,4

    24. group by

    25. group by 字段名
    26. 直接使用只取对应组的第一个
    27. 和聚合函数连用 AVG(), COUNT(), MAX(), MIN(), SUM()，计算组内结果
    28. 和where的区别：可以对各组进行单独求解

    29. 查询nobel表中2013-2015每年每科目获奖人数，按各年人数降序 

      select yr, subject, count(winner)
from nobel
where yr between 2013 and 2015
group by yr, subject
order by yr desc, count(winner) desc

    31. 对分组后数据进行筛选

    高级语句·

    1. 窗口函数
    2. over([partition by 字段][order by 字段 asc/desc])
    3. 排序窗口 rank()over(), dense_rank()over(), row_number()over()
    4. 偏移分析 lag(字段名,偏移量[,默认值])over(), lead(字段名,偏移量[,默认值])over() 往上/往下

    例题 - 对每年票数从高到低赋予名次 rank()over(partition by yr order by votes desc) as posn - covid每天新增人数

    1. 表连接

    2. from 表1 inner/left/right join 表2 on 表1.字段 = 表2.字段

    3. 内连接会筛掉没有连上的行
    4. 左/右连接保留对应 (from) 表内所有数据
    5. 重名时指定字段列所在表 select game.id
    select distinct t.name, d.name
from teacher t
left join dept d
on d.id = t.dept
    1. 子查询
    2. () 多层嵌套

    3. 一般用于from, where

    4. 组合查询

    5. union 分隔两条或以上的select语句
    6. 规则：每个查询必须包含相同列、表达式或聚集函数
    7. 场景：对一张表执行多个查询并按一个查询返回数据，或在一个查询中从不同表返回数据 
      select column_name(s) from table1
union
select column_name(s) from table2;
    8. union all 允许重复数据

    表·

    1. 创建
    2. create table 表名 (字段名 字段类型 约束) 
      create table student(
    id integer primary key autoincrement, --主键自增长
    name text not null, --非空
    age integer,
    email text unique, --唯一
    check (age>0)
);
    3. 参见Access, Server, MySQL各自数据类型规范。
    4. 约束
      • 主键 primary key 非空唯一
      • 非空 no null
      • 唯一 unique
      • 主键自增长（int时） autoincrement
      • 外键 foreign key 关联外表主键
      • check 限制列值范围
      • default 默认值
